Chapter 5: Managing Multiple Styles: The Cascade

A basic CSS mechanism known as the cascade governs how styles interact and which styles get precedence when there's a conflict.

In the case of conflict, the nearest ancestor wins. In other words, the style that's closest to the tag in question is the one that is applied.

Properties from a style specifically applied to a tag beat out any inherited properties.

Properties from the most specific style win.

Specificity formula

  1. Tag selector is worth 1 point.
  2. Class selector is worth 10 points.
  3. ID selector is worth 100 points.
  4. An inline style is worth 1000 points.

If after all of this there is still a tie, the last style wins.

!important; on any property in a declaration block shields it from specificity overrides. Note that !important applies to a specific property, not an entire style. 

  #nav a { color: red; }
  a { color: teal !important; }

The above makes all links on the page, including those inside an element with the #nav id, the color of teal.

Start with a clean slate

The author recommends erasing the built in browser formatting an apply your own. This is called "CSS reset" in the industry. 

  html, body, h1, h2, h3, h4, h5, h6, p, ol, ul, li, pre, code, address, variable, form, fieldset, blockquote {
    padding: 0;
    margin: 0;
    font-size: 100%;
    font-weight: normal;
  }
  ol {
    margin-left: 1.4em;
    list-style: decimal;
  }
  ul {
    margin-left: 1.4em;
    list-style:square;
  }
  img {
    border: 0;
  }
