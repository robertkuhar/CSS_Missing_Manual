Chapter 1: Rethinking HTML for CSS

The combination of HTML + CSS results in smaller pages and faster downloads.

HTML is the "structure" of your content. CSS is the style.

Simple HTML can increase your Search Page ranking

The <div> and <span> tags are like empty vessels that you fill with content

<div>-ination

A <div> is a block, meaning it has a line break before it and after it

A <div> indicates any discreet block of content...more often it's used to group any number of other elements, so you can insert a headline, a bunch of paragraphs, and a bulleted list inside a single <div> block.

A <div> is a great way to divide a page into logical areas, like 'banner', 'footer', 'sidebar', and so on. CSS can then be used to position each of these logical areas.

<span>-sters

A <span> appears inline as part of a paragraph

Treat <span> like any other inline elements like <a> or <strong>

HTML to forget

The 'hit list' includes: <font>, <b>, <i>

<table> is on shakey ground with him, use it only if you are presenting tabular data.

<body> loses most of its attributes

Forget about <br>

We will learn to rely upon CSS alone to control text placement, borders, backgrounds, and image alignment.

Rules of thumb

  1. Use only one <h1> per page
  2. Use <p> tag for paragraphs of text
  3. Use unordered lists <ul> to indicate steps in a process or define the order of a set of items.
  4. Glossaries of terms and definitions should be handled with 'definition list' and 'definition term'; <dt> and <dl> respectively.
  5. <blockquote> is appropriate for long quoted passages. <q> for one liners. The <blockquote> appears to work like a paragraph.
  6. <cite> for referencing a book title
  7. <address> for contact information and physical address.
  8. Validate your crap: the http://validator.w3.org or get a browser plug-in.
  9. Don't overuse <div>. Consider <ul> for the sidebar, for example.
  10. Be dilligent about closing tags.
