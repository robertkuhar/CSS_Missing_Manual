Chapter 4: Saving Time with Style Inheritance

Inheritance is the process by which some CSS properties applied to one tag are passed on to nested tags. This allows you to set a property in an ancestor, like <body>, and have it to the <p> tags and below. In other words, it doesn't only apply to the direct descendents, but their descendents as well.

I believe the operative term above is some properties.

The Limits of Inheritance

Inheritence isn't all-powerful. Many CSS properties don't pass down to descendent tags at all. The list below summarizes the most common cases where inheritence doesn't apply

  • Properties that affect the placement of elements on the page or the margins
  • Properties that affect background colors
  • Properties that affect borders
  • Web browser inherent styles (Headings are big and bold, for example). He recommends that you suppress these entirely. Chapter 5 covers this.
  • When styles conflict, the more specific style wins out. This is what allows you to style <li> elements with some specific formatting with a tag selector and it overcomes anything coming in via the <p> or above.

Note: Conflicts between styles are very common. The rules for how browsers deal with them are called the cascade.
