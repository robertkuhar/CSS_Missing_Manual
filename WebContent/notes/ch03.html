Chapter 3: Selectors: Identifying What to Style

Selectors in CSS allow you to specify what you want to format.

Tag Selectors: Page-Wide Styling

  • Tag Selectors are aka. "Type" or "Element" selectors
  • Efficient because they affect every occurrence of a tag on a page
  • "In essence, a tag selector redefines how a browser displays a particular tag."

Class Selectors: Pinpoint Control

You create a Class Selector by just giving it a unique name and some property/value attributes.

  • Class Selectors allow you to pinpoint the exact elements on a page you wish to affect regardless of the tag of those elements.
  • Unlike tag selectors, you can create as many class selectors as you like and put them anywhere on the page.
  • All class selector names must begin with a period. After the period the name must always start with a letter.
  • CSS permits only letters, numbers, hyphens, and underscores in class names
  • Class names are case sensitive.

To use a class style, you must add a class attribute to the HTML tag you wish to style (or to a <span> that you wrap the content you wish to style in)

  .special { font-weight: bold; }
  ...
  <p class="special">

ID Selectors: Specific Page Elements

The ID Selector identifies a specific part of the page to be affected by the format.

  • JavaScript can benefit from elements on the page having specific IDs
  • From a purely CSS viewpoint, IDs don't get you much that Class Selectors couldn't be used for.'

Rules of thumb when considering Class vs ID Selectors...

  • To use a style multiple times on a page, you must use Classes.
  • To use a style exactly once, for ever and ever, use IDs
  • Consider ID selector to sidestep style conflicts, since "web browsers give ID selectors priority over Class selectors.

Note: It is not valid HTML if an ID appears more than once on a page, the validators will fail, but most web browsers accept this fact and deal with it. 

  #banner {
      background: #CC0000;
      height: 300px;
      width: 720px;
  }
  ...
  <div id="banner">

<div> tags

A <div> tag identifies a logical division of the page, like a banner, navigation bar, sidebar, or footer. These are also called block-level elements because they form a complete "block" of conents (with line breaks before and after them). The <div> tag works just like a <p> tag.

A <div> tag has the unique ability to contain several block-level elements, making it a great way to group tags that are logically related (like logo and nav bar). Once grouped in this way, you can apply specific formatting to the tags inside the group.

Other reasons for IDs

IDs allow you to reference elements directly with Javascript rather than having to XPath or JQuery Selector locate them.

IDs also allow you to provide "jump here" hyperlinks (I thought only anchors did this). If this is true, clicking here should take us back to the <h2 id="id_selectors">ID Selectors: Specific Page Elements</h2> at the head of this section.

Group Selectors: Styling groups of Tags

Group Selectors allow you to apply a style to multiple selectors at the same time 

  h1, h2, h3, h4, h5, h6 { color: #F1CD33; }

There is nothing stopping you from having a Group Selector specify some of a tags format and then having an ID selector or Tag Selector provide other formatting. (I think they can even override each other, but the author has yet to explain the rules).

You can group dissimilar selectors in this way

  h1, p,.copyright, #banner { color: #F1CD33; }

The Universal Selector (Astertisk)

The '*' is special. It applies to every single tag. Sounds dangerous. 

  * { font-weight: bold; }

Descendent Selectors: Styling Tags with Tags

Descendent Selectors - Format a whole bunch of tags in a similar manner, but leave others of the same type alone. Descendent Selectors let you format a tab based upon its relationship to other tags. 

  h1 strong { color: red; }

The above has the effect of styling any text inside of <strong> tags red only if they are contained within an <h1> tag.

  ul li a { font-family: Arial; }

The above targets anchors within unordered lists.

Note: in general it is best to write as short a descendent selector as possible.

You are also not limited to just tag selectors when building your descendent selectors. Consider...

  p.intro a { color: yellow; }

...which turns <a> tags yellow that occur with <p> of class "intro"

Pseudo-classes and pseudo-elements

Pseudo-classes and pseudo-elements allow you to style some tags via well known attributes

Styles for Links

  • a:link - selects any link the user has not yet visited.
  • a:visited - selects any link the user has visited, according to the browser's history.
  • a:hover - allows you to change the look of the link as the mouse passes over. AKA. Rollover effect.
  • a:active - allows you to change the look of the link at the moment the user clicks and before he releases.

Note: You can use the :hover pseudo-class on elements other than links, but beware IE when not in standards mode.

BEYOND THIS POINT, the Selectors become less useful. Some are rarely used, mostly because IE doesn't support them in any consistent manner until IE8 and beyond.

  • p:first-letter - :first-letter pseudo-element lets you create a drop cap, an initial letter that jumps out from the rest of the paragraph with bigger or bolder font formatting, as at the beginning of a book chapter.
  • p:first-line - :first-line of a paragraph in a different color keeps your reader's eye moving and makes text appear appealing and fresh

:before and :after

The :before pseudo-element allows you to add content before some tag; :after allows it after. Doesn't work in IE until IE8..

:first-child

The :first-child pseudo-element allows you to select just the first of "sibling" tags for formatting. 

  li:first-child { font-weight: bold; }

:focus

The :focus pseudo-class works a lot like the :hover pseudo-class above.

The :focus selector is mostly useful for giving your visitor feedback, like changing the background color of a text box to indicate where he's about to type. The following adds a light yellow color to any text box a visitor clicks or tabs into...

  input:focus { background-color: #FFFFCC; }

Child Selectors

Child Selectors are like Descendent Selectors, only more selective. An angle bracket '>' indicates the exact relationship between the parent and the child. 

  body > h2 { font-weight: bold; }

The child selector above would select <body><h2> but not <body><div><h2>. A descendent selector would have got them both.

Adjacent Sibling Selectors

Adjacent Sibling Selectors work on the relationship between 2 or more tags that are peers of each other in the tree, rather than children as in Child Selectors and Descendent Selectors. 

  h2 + p { font-weight: bold }

The Adjacent Sibling selector above formats just "This is P1" in <h2>Heading</h2><p>This is P1</p><p>This is P2</p>.

Attribute Selectors

CSS provides a way to select tags for formatting using their attributes (like an <img> tag's title attribute).

  img[title] { border-style: solid; border-width: 1px; }

The above Attribute Selector will format all the image tags that have a title attribute. 

  a[href="http://www.cosmofarmer.com"] { color: red; font-weight: bold; }

The above Attribute Selector will format all the anchor tags that have an href attribute with a value of http://www.cosmofarmer.com. The "attribute equals" portion can be somewhat regex-y as well like... 

  a[href$=".pdf"] { background-image: url(doc_codon.png) no-repeat; padding-left: 15px; }

...which does some formatting to all the anchors with an href that ends in .pdf 

  img[src*="headshot"]

Selects all images which has the word "headshot" somewhere in the file

Note: He never says that the attribute specifiers are actually regex, they just have a couple of similarities.
