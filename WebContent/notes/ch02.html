Chapter 2: Creating Styles and Style Sheets

2.1. Anatomy of a Style

A style is made up of two elements:

  1. Selector - the web page element that the browser formats
  2. Declaration Block - the actual formatting instructions
p { color: red; font-size: 1.5em;}

...tells the browser "make the text in all paragraphs red and 1.5 ems tall". Note that "em" is a unit of measure based upon a browser's normal text size.

Selector

Selector tells the browser which element or elements on a page to style. Chapter 3 covers selectors in depth.

Declaration Block

Includes all formatting options you want to apply to the element(s) identified by the Selector. Declaration Blocks are made up of one or more "Declarations".

Declaration

Each Declaration is made up of a "Property" and a "Value". The "Property" is sepearted from the "Value" by a colon ':'. Each Declaration is separated from the other Declarations by a semi-colon ';'.

In the above example the Declaration Block has 2 Declarations.

  1. color: red;
  2. font-size: 1.5em;
Property

A Property is a word, or a few hyphenated words, indicating a certain style effect.

Common properties are: font-style, margin-top, text-align, background-color.

Value

The Property dictates the range of legitimate values here. The Property defines what category of style we are configuring, the Value contains the specifics. Ex. background-color: blue;

2.2. Understanding Style Sheets

A collection of CSS styles comprises a style sheet. There are 2 types of sytle sheets.

  1. Internal - Style information is located in the web page itself.
  2. External - Style information is located in a separate file and linked in the head.

External sheets are usually the way to go, just don't get caught up in cache problems. Forcing a page reload refetches the extenal style sheets. Command-R on Mac. F5 everywhere else, I think.

Internal style sheets always appear with <style> tags in the page's <head> section.

The author has an interesting note about <style> relative to <script> tags. He recommends that <style> goes first so that the scripts can work with them.

ALWAYS END AN EXTERNAL SHEET'S FILENAME WITH .css.

There are 2 ways to link an external stylesheet with a page:

  1. <link rel="stylesheet" type="text/css" href="mystyle.css"/>
  2. <style type="text/css">
    @import url(mystyle.css);
    </style>

The @import thing works within a stylesheet as well.
